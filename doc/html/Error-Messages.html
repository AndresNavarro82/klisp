1.3.4 Error Messages

Some examples cause errors to be signaled. The report doesn't specify what objects are passed to the error continuation, but in klisp, objects passed to the error continuation are encapsulated error objects that have at least a message and possibly some additional objects and context informations (such as source code location). In the examples, the error message is shown on a line starting with ‘error-->’. 

     (+ 23 #t)
     error--> Wrong type argument: (expected number) (#t)
